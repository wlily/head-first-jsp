• ServletConfig vs ServletContext
    • One servlet one ServletConfig
      • getInitParameter
      • getServletContext
    • One web app one ServletContext
      • getInitParameter
      • getAttribute
      • setAttribute
      • removeAttribute
      • getRequestDispatcher
  • Listener
    • ServletContextListner
      • Get notified when the context is initialized(app deployed)
      • Get notified when the context is destroyed(app undeployed or goes down)
    • ServletContextAttributeListener
    • ServletRequestListener
    • ServletRequestAttributeListener
    • HttpSessionListener
    • HttpSessionBindingListener
    • HttpSessionAttributeListener
    • HttpSessionActivationListener
  • Attributes vs Parameters
    • Attributes
      • Types:Application/context, Request, Session
      • Set Method:setAttribute(String,Object)
      • Return Type:Object
      • Get Method:getAttribute(String)
    • Parameters
      • Types:Application/context init parameters, Request parameters, Servlet init parameters
      • Set Method:No
      • Return Type:String
      • Get Method:getInitParameter(String),getParameter(String)
  • Thread Safe Issues
    • Unsafe
      • ServletContext Attributes
      • Session Attributes
      • Servlet Instance Variable
    • Safe
      • Request Attributes
  • Dispatch Issues
    • Request attributes make sense when you want some other component of the app to take over all or part of the request. eg. MVC.
    • RequestDispatcher can make another part of the component take over the request.
    • Get a RequestDispatcher from a ServletRequest or a ServletContext.
    • Calling forward() on RequestDispatcher.
